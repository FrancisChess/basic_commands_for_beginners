Comandos Básicos do Git

Este documento contém uma lista de comandos básicos do git úteis para iniciantes.

  1. mkdir [nome_pasta]
    2. criar uma pasta
  2. ls
    3. Apresenta os diretorios (pastas)

  3. git init
    4. Inicializa um repositório

  4. cd ..
    5. Sai da pasta

  5. cd "nome_da_pasta"
    6. Entra em uma pasta/documento/diretório

  6. ls -lf
    7. Apresenta arquivos ocultos

  7. ls "nome_arquivo/pasta"
    8. Apresenta arquivos em uma pasta

  8. touch "nome_arquivo"
    9. Cria um arquivo vazio

  9. git status
    10. Apresenta o status do repositório

  10. git add "nome_arquivo"
    11. Para enviar arquivos para um commit

  11. git restore "nome_arquivo"
    12. Para restaurar uma versão

  12. git diff
    13. Apresenta as mudanças

  13. git commit -m "Mensagem"
    14. Confirma a modificação dos arquivos

  14. git log
    15. Observar os commits realizados no projeto

  15. git clone "link_github"
    16. Para clonar o projeto do GitHub na pasta selecionada

  16. git log --oneline --all
    17. Versão simplificada do log

  17. code .
    18. Abre o VSCode

  18. git push origin master
    19. Envia as atualizações para o repositório remoto (origin) na branch master

  19. git pull origin master
    20. Recebe as atualizações do repositório remoto (origin) na branch master

  20. git branch "nome_branch"
    21. Cria uma branch

  21. git branch
    22. Mostra as branches

  22. git checkout "nome_branch"
    23. Vai para outra branch

  23. git checkout -b "nome_branch"
    24. Cria uma branch e já muda para a branch criada

  24. git merge "nome_branch"
    25. Faz com que mescle as modificações de uma branch na branch atual

  25. git restore
    26. Restaura os últimos arquivos não commitados

  26. git reset --soft "commitId"
    27. Volta para o estágio antes do commit

  27. git reset --mixed "commitId"
    28. Volta os itens antes do staged (antes do git add)

  28. git reset --hard "commitId"
    29. Volta para os antigos arquivos e apaga as alterações realizadas

  29. git restore --staged "fileName"
    30. Retira o arquivo do staged

  30. git commit --amend -m "mensagem"
    31. Junta o commit com o commit anterior

  31. clear
    32. Limpa o bash
